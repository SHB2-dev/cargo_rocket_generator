セットアップ手順

        
Ad Astra Cargo Rockets と CC:Tweaked を使って自動化を始めるための手順です。

        
必要なMOD

        

          
1
Ad Astra
宇宙MOD本体。ロケットや惑星を追加します。Fabric 1.20.1 対応版を導入してください。

          
2
Ad Astra Cargo Rockets
CC:Tweaked でロケットを自動制御できるようにするアドオン。CurseForge から導入。

          
3
CC:Tweaked
ComputerCraft の後継MOD。コンピュータブロックと Lua スクリプトを追加します。

          
4
Fabric API + F-Tech: Core
Cargo Rockets の動作に必要な依存MODです。

        

        
ブロックの設置方法

        

          
1
Cargo Rocket Launch Pad を設置
発射台を設置します。中央ブロックと外周ブロックで構成されています。

          
2
PCを発射台の中央ブロックに隣接させる
⚠️ 外周ブロックではなく中央ブロックに隣接させてください。外周だとAPIが使えません。Peripheral Hub 経由でも可。

          
3
接続確認
PCを開いて lua と入力後、以下を実行して発射台が見つかることを確認します。

        

        
peripheral.getNames()
-- 例: { "top" } と表示されればOK

launch_pad = peripheral.wrap("top")
launch_pad.isRocketPresent()
-- true / false が返ればAPI接続成功

        
PCと発射台が離れている場合はPeripheral Hub を使って有線接続できます。その場合もHubは中央ブロックに隣接させてください。

      


      
      

        
スクリプトの使い方

        
生成したLuaスクリプトをComputerCraftのPCに保存して実行する方法です。

        
ファイルの保存方法

        
方法 A — PC内で直接書く

        
PCを開いてシェルから以下を実行します：

        
edit startup.lua

        
エディタが開くので、ジェネレーターでコピーしたコードを貼り付けて Ctrl+S で保存、Ctrl+E で終了します。

        
方法 B — pastebin を使う（推奨）

        
pastebin.com にコードを貼り付けてIDを取得し：

        
pastebin get <ID> startup.lua

        
実行方法

        

          
1
手動実行
PCシェルで startup と入力して Enter。

          
2
自動起動 (startup.lua)
ファイル名を startup.lua にするとPC起動時に自動実行されます。再起動は reboot コマンドで。

          
3
停止方法
Ctrl+T でプログラムを強制終了できます。

        

        
スロット番号は1始まりです。発射台スロット1〜9はインプットスロット（ホッパー等が入れる側）なので、スクリプトで積み込む際は10以降を使うと混乱しにくいです。

ティア計算のルール

        
必要ティア = max(発射地点のティア, 目的地のティア)

        
-- 例1: 地球 → 火星  max(0, 2) = Tier 2
-- 例2: 火星 → 地球  max(2, 0) = Tier 2
-- 例3: 月  → 地球軌道  max(1, 1) = Tier 1

        
ジェネレーターの「発射地点」と「目的地」を選ぶと必要ティアが自動計算されます。

      


      
      

        
Lua API 一覧

        
launch_pad = peripheral.wrap("top") で接続後に使用できる全コマンドです。

        
発射・状態確認

        

          
関数戻り値説明

          
            
launch(planet)—指定した惑星へロケットを発射。失敗するとエラーを投げる。

            
isRocketPresent()boolean発射台にロケットがあれば true。

            
getValidDestinations()table行ける惑星と必要ティアの一覧。例: { ["ad_astra:moon"] = 1 }

          
        

        
エネルギー

        

          
関数戻り値説明

          
            
getEnergy()long現在の蓄積エネルギー量 (RF)。

            
getMaxEnergy()long最大エネルギー容量 (RF)。

            
getEnergyRequiredForLaunch()int発射に必要なエネルギー量。通常 500,000 RF。

          
        

        
インベントリ操作

        

          
関数戻り値説明

          
            
listRocketInventory()table|nilロケット内のアイテム一覧。ロケットがなければ nil。

            
listLaunchPadInventory()table発射台インベントリのアイテム一覧。

            
moveItemsFromLaunchPadToRocket(padSlot, rocketSlot)—発射台 → ロケットへアイテムを移動。

            
moveItemsFromRocketToLaunchPad(rocketSlot, padSlot)—ロケット → 発射台へアイテムを移動。

            
listLaunchPadInputSlotIndexes()int[]ホッパー等が挿入できるインプットスロット番号一覧。

            
listLaunchPadOutputSlotIndexes()int[]ホッパー等が取り出せるアウトプットスロット番号一覧。

          
        

      


      
      

        
トラブルシューティング

        
よくあるエラーと解決方法をまとめました。

        
接続エラー

        

          
attempt to index global 'launch_pad' (a nil value)
原因： peripheral.wrap() が未実行、またはPC再起動で変数がリセットされた。
解決： スクリプト先頭に local launch_pad = peripheral.wrap("top") を必ず記述。

          
peripheral.getNames() が { "bottom" } しか返らない
原因： PCがモデムやHub に接続しているが発射台が認識されていない。
解決： peripheral.getType("bottom") で種類を確認。HubなどにPCを繋げ直す。

          
attempt to call field 'isRocketPresent' (a nil value)
原因： PCが発射台の外周ブロックに接続している。外周ではAPIが使えない。
解決： PCまたはHubを発射台の中央ブロックに隣接するよう設置し直す。

        

        
発射エラー

        

          
No rocket found
原因： 発射台にロケットが設置されていない。
解決： ロケットを発射台の中央ブロックに設置する。

          
Not enough energy to launch
原因： エネルギー不足（通常 500,000 RF 必要）。
解決： 発電機でエネルギーを供給。launch_pad.getEnergy() で現在量を確認。

          
<planet> is too high of a tier for this rocket
原因： ロケットのティアが目的地に対して不足している。
解決： より高ティアのロケットを使用する。ジェネレーターのティア計算で確認。

          
<planet> is not a valid planet
原因： 惑星IDが間違っている（タイポなど）。
解決： launch_pad.getValidDestinations() で正しいIDを確認。

        

        
よくある質問

        

          
スロット1〜9にアイテムを入れたのにロケットに積まれない
スロット1〜9は発射台のインプットスロットです。積み込みには10番以降のスロットを使ってください。

          
スクリプトを止めたい
PCを開いた状態で Ctrl+T を押すとプログラムを強制終了できます。

          
再起動したらスクリプトが動かない
ファイル名を startup.lua にすることでPC起動時に自動実行されます。
